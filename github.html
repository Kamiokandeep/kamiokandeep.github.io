GitHub Python C#

GitHub, an essential platform in software development, utilizes the Git version control system to facilitate collaboration among developers. By providing repositories to store projects, GitHub enables precise tracking of changes and streamlines the integration of contributions through pull requests.

In addition to features like issues and wikis, GitHub stands out for its automation capabilities with actions. This platform has evolved to become a comprehensive hub fostering transparency and community in software development.

Settings

>>> git config --global user.name "User Name"

This command globally sets the username associated with commits in Git. Replace "User Name" with your Git username. It is crucial for identifying who made changes in the version history.

>>> git config --global user.email "User Email"

This command globally configures the email address associated with commits in Git. Replace "User Email" with your email address linked to your Git account. It is essential for author identification in the version history.

First Steps

>>> git init

Initializes a new Git repository in the current directory.

>>> git add file.txt

Adds the specified file to the staging area to be included in the next commit.

>>> git add *

Adds all files in the current directory to the staging area.

>>> git commit -m "msg"

Commits changes with a descriptive message.

>>> git status

Shows the current status of files in the repository, highlighting pending changes.

>>> git pull

Fetches the latest changes from the remote repository and merges them into the local repository.

>>> git push

Pushes local changes to the remote repository.

In summary

>>> git init
>>> git add file.txt
>>> git commit -m "msg"
>>> git pull
>>> git push

Log

>>> git log file.txt

Shows the commit history specifically for the specified file.

>>> gitk

Is a graphical interface for visualizing the commit history and branches in Git.

>>> git show file.txt

The command displays detailed information about changes made to a specific file. It provides the content of the file in the most recent commit that affected it, along with metadata such as the author, date, and commit message. It is useful for reviewing changes and understanding the evolution of a file in the version history.

>>> git diff tag_1 tag_2

The command shows the differences between two points in Git history, identified by tags or any other object representing a commit. It provides a detailed view of the changes between the corresponding versions, aiding in understanding the modifications made.

Delete

>>> git rm file.txt

Remove a file from both the working directory and the staging area simultaneously. This means the file will be deleted both locally and in the next commit.

>>> git rm --cached file.txt

Removes the file from the staging area but keeps it in the working directory. This indicates that the file will be deleted from version control tracking but will remain in your local file system.

In summary

>>> git add file.txt
>>> git status
>>> git rm file.txt
>>> git status
