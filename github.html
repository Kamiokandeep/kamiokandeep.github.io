Welcome to GitHub Notes!

Posted on January 1, 2024
GitHub Python C# HTML CSS

GitHub, an essential platform in software development, utilizes the Git version control system to facilitate collaboration among developers. By providing repositories to store projects, GitHub enables precise tracking of changes and streamlines the integration of contributions through pull requests.

In addition to features like issues and wikis, GitHub stands out for its automation capabilities with actions. This platform has evolved to become a comprehensive hub fostering transparency and community in software development.

Settings

>>> git config -l

The git config -l command displays the current Git configuration, including user information, formatting options, and other settings. It is useful for reviewing and verifying the current configuration in your Git environment.

>>> git config --global user.name "User Name"

This command globally sets the username associated with commits in Git. Replace "User Name" with your Git username. It is crucial for identifying who made changes in the version history.

>>> git config --global user.email "User Email"

This command globally configures the email address associated with commits in Git. Replace "User Email" with your email address linked to your Git account. It is essential for author identification in the version history.

First Steps

>>> git init

Initializes a new Git repository in the current directory.

>>> git add file.txt

Adds the specified file to the staging area to be included in the next commit.

>>> git add *

Adds all files in the current directory to the staging area.

>>> git commit -m "msg"

Commits changes with a descriptive message.

>>> git status

Shows the current status of files in the repository, highlighting pending changes.

>>> git pull

Fetches the latest changes from the remote repository and merges them into the local repository.

>>> git push

Pushes local changes to the remote repository.

In summary

>>> git init
>>> git add file.txt
>>> git commit -m "msg"
>>> git pull
>>> git push

Using a shortcut

>>> git commit -am "msg"

The git commit -am "msg" command performs two actions in a single step. The -a flag automatically adds all modified files to the staging area, while -m "msg" allows you to specify the commit message on the command line. This streamlines the process of committing changes that are already being tracked in the repository.

In summary

>>> git init
>>> git commit -am "msg"
>>> git pull
>>> git push

Log

>>> git log file.txt

Shows the commit history specifically for the specified file.

>>> gitk

Is a graphical interface for visualizing the commit history and branches in Git.

>>> git show file.txt

The command displays detailed information about changes made to a specific file. It provides the content of the file in the most recent commit that affected it, along with metadata such as the author, date, and commit message. It is useful for reviewing changes and understanding the evolution of a file in the version history.

>>> git diff tag_1 tag_2

The command shows the differences between two points in Git history, identified by tags or any other object representing a commit. It provides a detailed view of the changes between the corresponding versions, aiding in understanding the modifications made.

Delete

>>> git rm file.txt

Remove a file from both the working directory and the staging area simultaneously. This means the file will be deleted both locally and in the next commit.

>>> git rm --cached file.txt

Removes the file from the staging area but keeps it in the working directory. This indicates that the file will be deleted from version control tracking but will remain in your local file system.

In summary

>>> git add file.txt
>>> git status
>>> git rm file.txt
>>> git status

Reset

>>> git reset tag --soft

The command performs a "soft" reset, keeping local changes in the staging area. The branch and working directory are updated to the state of the commit referenced by the tag, but uncommitted local changes are retained so that they can be included in the next commit.

>>> git reset tag --hard

The current branch is reset to the state of the commit referenced by the tag. Local changes and those in the staging area are completely discarded, and the branch and working directory are updated to the state of the commit indicated by the tag.

Restore File

>>> git checkout tag file.txt

You can restore the specific content of a file (file.txt) to the state associated with the tag (tag). This command is useful when you want to view or revert the state of a file to a specific version identified by the tag in the repository's history.

In summary

>>> git log file.txt
>>> git checkout tag file.txt
>>> git add file.txt
>>> git commit -m 'msg'
>>>

In summary, git checkout tag file.txt will restore the file in your working directory, and then you need to use git add and git commit to include those changes in the staging area and make a new commit, respectively.

Branches

>>> git branch

The git branch command displays a list of all branches in your local repository. The current branch is highlighted with an asterisk (*). It is useful for identifying existing branches and determining which branch you are currently on.

>>> git branch "new_branch_name"

The git branch "new_branch_name" command is used to create a new branch in Git with the specified name. However, creating the branch does not change your current position; you remain in the current branch.

>>> git checkout "new_branch_name"

Using the git checkout "new_branch_name" command, you switch your position to the specified new branch. Now, you will be working on that branch, and any changes you make will apply to that branch instead of the current branch.
